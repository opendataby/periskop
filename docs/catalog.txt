# Каталог показателей

Просто и без изысков: поиск показателя, вывод данных на график, (сохранение данных к себе в систему), возможность сформировать свой рабочий стол с избранными показателями.
Начинать без роутера. Проще - лучше.

Хозяева показателей - заинтересованные стороны. "Битва показателей".

Сделать до 10000 показателей, разместить индекс на Алголии. ТН ВЭД не размещать, сделать субпроектом.

## Государственный долг, всего

Сценарий

1. На главной странице формируется стол из Н графиков под строкой поиска.
2. Ввод в строку поиска названия показателя выдает список совпадений с возможностью выбора показателя
3. После выбора показателя, соотв. данные в виде графика добавляются сверху к уже имеющимся
4. У каждой карточки (график) есть крестик для удаления со стола
5. Формат стола: 2 колонки из графиков
6. Данные по показателям хранятся на FB, индекс - на своеь API.

{
	"12": {
		"title": "Государственный долг, всего",
		"dim": "BYN млрд",
		"data": []
	},
	"13": {
		"title": "Численность населения, на ук. дату",
		"dim": "человек",
		"data": []
	}
}


Легенда для ключей в данных. Ключи однобуквенные для экономии объема передаваемых данных:

- t - title, Название показателя
- l - label, ярлык для оси и единица измерения показателя
- c - chart, тип графика (столбиковый, линейный ...)
- d - data, сами данные
	-- t - time, временная привязка (отчетный период или дата)
	-- a - amount, числовое значение показателя на указанную дату

Реструктурировать данные для recharts:

[ {"101": {"t": "Численность населения, на ук. дату", "l": "человек", "c": "bar", "d": [{"t": 2009-12-01, "a": 23840.12}]}, ... ]
